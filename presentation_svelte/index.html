Introduction

C'est quoi Svelte ?

Svelte est un compilateur, pas un framework classique. Il transforme vos composants en code JavaScript impératif ultra-optimisé au moment du build. Zero runtime envoyé au navigateur.

Créé par Rich Harris (NY Times) en 2016. Maintenant chez Vercel.

Code .svelte
HTML + CSS + JS
dans un seul fichier
→
Compilateur Svelte
Analyse & optimise
au build time
→
JS Vanilla optimisé
Pas de runtime !
DOM direct
Feature #1

Le framework qui disparaît

Svelte compile votre code. Au runtime, le framework n'existe plus — il ne s'interpose jamais entre votre code et l'utilisateur.

React / Vue (Virtual DOM)

1. État change
↓
2. Re-render du composant entier
↓
3. Créer nouvel arbre VDOM
↓
4. Diff ancien vs nouveau
↓
5. Patch le vrai DOM
Runtime nécessaire (~40-65 KB)

Svelte (Compilation)

1. État change
↓
2. Met à jour exactement le nœud DOM concerné
0 KB de runtime — JS vanilla pur

Pas de diffing, pas de réconciliation.
Le compilateur sait à l'avance quoi mettre à jour.

Svelte & la Performance Web

Ton cours explique pourquoi le JS lourd est un problème. Svelte est une réponse architecturale à ce problème.

CRP — Script bloquant
Le navigateur bloque le DOM à chaque <script>.
Angular → 65 KB → ~300ms de blocage
Svelte → 2 KB → ~10ms de blocage
30× moins de blocage CRP.
Look-ahead parser
Le navigateur anticipe les ressources critiques à télécharger.
Il priorise les <script> — plus ils sont légers, plus vite ils passent.
Svelte tire pleinement parti de cette optimisation.
Layout & Paint pipeline
JS → Style → Layout → Paint → Composite
Chaque modif DOM peut déclencher un reflow complet.
React VDOM → patches multiples → reflows inutiles
Svelte → update chirurgicale → moins de reflows → meilleur FPS
Modèle RAIL (Google)
Load <5s — moins de KB → plus facile à tenir
Response <50ms — pas de VDOM → interactions plus rapides
Animation 10ms/frame — moins de JS → plus de CPU libre
Svelte s'aligne naturellement sur RAIL.
 "40% des utilisateurs quittent un site qui met plus de 3 secondes à charger." La différence Angular→Svelte (~290ms sur mobile) peut faire basculer ce seuil.
Feature #3

Code & Réactivité

Réactivité native, CSS scopé, animations intégrées, 40% de code en moins.

SVELTECounter.svelte 
<script>
  let count = $state(0);
  let double = $derived(count * 2);
</script>

<button onclick={() => count++}>
  {count} (double: {double})
</button>

<style>
  button { color: red }
</style>
REACTCounter.jsx 
import {useState, useMemo}
  from 'react';

export default function Counter() {
  const [count, setCount] =
    useState(0);
  const double = useMemo(
    () => count * 2, [count]
  );
  return (
    <button onClick={() =>
      setCount(count + 1)}>
      {count} (double: {double})
    </button>
  );
}
VUECounter.vue 
<script setup>
import { ref, computed }
  from 'vue';

const count = ref(0);
const double = computed(
  () => count.value * 2
);
</script>

<template>
  <button @click="count++">
    {{ count }} ({{ double }})
  </button>
</template>
$state — état réactif
$derived — calculé automatique
CSS scopé — isolé par composant

Feature #4

Svelte Stores

Un state management global intégré — sans Redux, sans Pinia, sans lib externe.

writable

Store en lecture/écriture. Le plus courant.

import { writable } from
  'svelte/store';

const count = writable(0);

count.set(5);
count.update(n => n + 1);

readable

Store en lecture seule. Idéal pour des données externes.

import { readable } from
  'svelte/store';

const time = readable(
  new Date(),
  set => {
    const i = setInterval(()
      => set(new Date()), 1000);
    return () => clearInterval(i);
  });

derived

Calculé à partir d'autres stores. Auto-synchronisé.

import { derived } from
  'svelte/store';

const doubled = derived(
  count,
  $count => $count * 2
);
Auto-souscription avec le préfixe $store
Partagé entre composants sans props drilling
0 KB de lib supplémentaire

Svelte en Web Components

Svelte peut compiler vos composants en Web Components standards — utilisables dans React, Vue, Angular ou du HTML pur.

Perspective

Svelte à l'ère de l'IA

L'IA ne remplace pas les frameworks — elle les rend encore plus importants.

Interfaces IA légères

Chatbots, dashboards IA, outils interactifs — Svelte produit des bundles minimaux, idéal pour des UI rapides autour de modèles IA.

IA génère du code Svelte

La syntaxe simple de Svelte est plus facile à générer par les LLMs. Moins de boilerplate = moins d'erreurs de génération.

Compilation = optimisation auto.

Même du code généré par IA est optimisé par le compilateur. Le résultat est toujours performant.

Prototypage rapide

Avec l'IA + Svelte, on peut itérer très vite : moins de code à écrire, plus de temps pour l'expérimentation.

Construire une Todo App

Étape par étape — découvrez la puissance de Svelte en pratique

Structure d'un composant

Un fichier .svelte combine HTML, CSS et JS dans un seul fichier.

Pas d'import React, pas de JSX, pas de return(). Juste du code naturel.

Le CSS est automatiquement scopé au composant.
SVELTEApp.svelte 
<script>
  let name = 'Svelte';
</script>

<h1>Bonjour {name} !</h1>

<style>
  h1 { color: #ff3e00 }
</style>

Réactivité avec $state

$state() déclare un état réactif. $derived() calcule une valeur dérivée.

Pas de useState, pas de setter. Juste une variable qui trigger les updates.
SVELTETodo.svelte 
<script>
  let todos = $state([]);
  let newTodo = $state('');
  let remaining = $derived(
    todos.filter(t => !t.done).length
  );
</script>

<h2>{remaining} tache(s)</h2>

Événements et binding

bind:value = two-way binding. onclick = event handler.

bind:value remplace onChange + setState de React.
SVELTETodo.svelte 
<input
  bind:value={newTodo}
  placeholder="Nouvelle tache..."
  onkeydown={e =>
    e.key === 'Enter' && addTodo()}
/>
<button onclick={addTodo}>
  Ajouter
</button>

Boucles avec {#each}

{#each} pour iterer, {#if} pour conditionner.

Pas de .map() avec return JSX. Template déclaratif.
SVELTETodo.svelte 
{#each todos as todo (todo.id)}
  <div class:done={todo.done}>
    <input type="checkbox"
      bind:checked={todo.done} />
    <span>{todo.text}</span>
  </div>
{:else}
  <p>Aucune tache !</p>
{/each}

Transitions & animations

Svelte intègre un système d'animations natif. Pas de librairie externe.

En React : framer-motion (30+ KB). En Svelte : natif et 0 KB supplementaire.
SVELTETodo.svelte 
<script>
  import { slide, fade }
    from 'svelte/transition';
  import { flip }
    from 'svelte/animate';
</script>

{#each todos as todo (todo.id)}
  <div
    transition:slide={{ duration: 300 }}
    animate:flip={{ duration: 200 }}
  >
    {todo.text}
  </div>
{/each}

REACTTodo.jsx — 42 lignes 
import {useState,useMemo}
  from 'react';

export default function Todo() {
  const [todos,setTodos] =
    useState([]);
  const [text,setText] =
    useState('');
  const left = useMemo(=>
    todos.filter(t=>!t.done).length,
    [todos]);

  function add() {
    if(!text.trim()) return;
    setTodos([...todos,
      {text, done:false,
       id:Date.now()}]);
    setText('');
  }
  function toggle(id) {
    setTodos(todos.map(t =>
      t.id===id ? {...t,
        done:!t.done} : t));
  }
  return (<>
    <h2>Todos ({left})</h2>
    <input value={text}
      onChange={e=>setText(
        e.target.value)} />
    {todos.map(t=>
      <label key={t.id}>
        <input type="checkbox"
          checked={t.done}
          onChange={()=>toggle(t.id)}/>
        {t.text}
      </label>)}
  </>);
}

SSR + CSR + SSG

API intégrée

+server.js pour les endpoints REST. Backend + frontend dans le même projet.

Déploiement universel

Vercel, Netlify, Cloudflare, Node.js, static — adaptateurs pour tout.
